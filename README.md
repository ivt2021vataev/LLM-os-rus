# Обучение GPT-модели для Осетинского Языка (124M)

Репозиторий содержит генеративную языковую модель на базе классической архитектуры GPT-2 (124M параметров), обученную с нуля на базе Осетинско-Русского словаря (издание 1970 года). 

Главная особенность проекта — использование кастомного BPE-токенизатора, оптимизированного под кириллицу и специфические символы осетинского алфавита (включая букву **æ**), что предотвращает появление артефактов кодировки (ромбиков ``) при генерации.

## Файлы в репозитории

| Файл | Описание |
| :--- | :--- |
| `model.py` | Архитектура нейросети (LayerNorm, MultiHeadAttention, GPTModel) и метод инференса. |
| `create_tokenizer.py` | Скрипт сборки и обучения кастомного BPE-токенизатора на базе текстового корпуса. |
| `train.py` | Оптимизированный цикл обучения модели с контролем градиентов и кастомным даталоадером. |
| `test.py` | Скрипт для тестирования обученной модели и генерации текста по промпту. |
| `os_ru_dataset.txt` | Подготовленный текстовый датасет на основе словаря (~21 000 словарных статей). |

## Зачем нужен кастомный токенизатор?

Стандартный английский токенизатор `tiktoken` (GPT-2) не знаком с осетинским языком и кириллицей. Он дробит каждую букву на цепочки отдельных байтов. Из-за этого модель на ранних этапах обучения путает байты, порождая сломанные символы ``, а длина контекста искусственно раздувается в 4–5 раз.

Собственный **BPE (Byte Pair Encoding)** токенизатор, обученный прямо на `os_ru_dataset.txt`, решает эти проблемы:
* Словарь ограничен **15 000 токенов** (оптимально для данного объема данных).
* Осетинские слова, частые слоги и буква **æ** становятся цельными токенами.
* Полностью исключены ошибки декодирования символов.
* Скорость утилизации контекста моделью возрастает в разы.

## Структура датасета

Текстовый корпус `os_ru_dataset.txt` содержит 21 005 строк, отформатированных в виде естественного текста для языковой модели:
```text
Осетинское слово «абон» означает «нареч. времени сегодня; абон уын рассвести...».
Осетинское слово «аба кæнын» означает «поцеловать.».
Такой формат позволяет модели на этапе инференса легко цепляться за промпт типа Осетинское слово «[слово]» и продолжать его точным переводом.

Параметры модели (GPT_CONFIG_124M)
vocab_size: 15 000 (размер кастомного словаря)
context_length: 1024 (в даталоадере используется скользящее окно на 256 токенов)
emb_dim: 768 (размерность эмбеддингов)
n_heads: 12 (количество голов внимания)
n_layers: 12 (количество трансформер-блоков)
drop_rate: 0.1
qkv_bias: False

Требования к запуску
Python 3.10 или новее
PyTorch (рекомендуется сборка с поддержкой CUDA для обучения на GPU)
Hugging Face tokenizers

Установка библиотек:
pip install torch tokenizers

Порядок работы с проектом
1. Сборка токенизатора
Перед запуском обучения необходимо собрать словарь токенов на основе вашего датасета:
python create_tokenizer.py
В папке появится файл os_ru_tokenizer.json.

2. Запуск обучения
Скрипт автоматически подтянет токенизатор, переведёт текст в ID и запустит обучение на 
доступном устройстве (CUDA/CPU). В цикле применяется ограничение градиентов (clip_grad_norm_) 
для защиты от взрыва лосса на повторяющихся текстовых шаблонах:
python train.py
Итоговые веса запишутся в файл model_final.pth.

3. Тестирование генерации
Скрипт загружает веса, переводит текстовый промпт в токены и запускает пошаговую генерацию продолжения:
python test.py
